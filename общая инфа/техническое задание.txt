развернутый сайт будет состоять из нескольких контейнеров:
    контейнер с сервером php
    контейнер с микросервисом на python
    контейнер qdrant

типичный сценарий работы сервиса:
    1. пользователь открыл сайт в своем браузере (попутно офигел с сногшибательной красоты сайта)
    2. пользователь вбил в текстовое поле запрос для поиска документов и нажал на кнопку "поиск"
    3. функция JavaScript выполнила асинхронный запрос на php сервер с помощью AJAX (функция отправила поисковой запрос, а ожидает получить ссылку на статью или несколько статей на хабре)
    4. php сервер получил запрос от JavaScript функции
    5. php сервер отправил python микросервису через Curl запрос поисковой запрос
    6. python микросервис принял запрос от php сервера с помощью FastAPI
    7. python микросервис посчитал эмбеддинг поискового запроса.
    7. python микросервис отправил qdrant контейнеру эмбеддинг поискового запроса
    8. qdrant нашел близкий к эмбеддингу поискового запроса эмбеддинг и ссылку на соответсвуюю ему статью
    9. qdrant вернул python скрипту ссылку на статью
    10. далее ссылка идет назад тем же путем через сервисы пока не покажется у пользователя на экране (важно чтобы пользователью вывелся результат без перезагрузки странички)




нерешенные/непонятные вопросы:
    как сделать чтобы база данных "эмбеддинги - ссылки" была внутри qdrant контейнера?
        (ну или как она должна подключатся через volumes)

    пользователью должна возвращатся ссылка на одну или на несколько подходящих статей?


уровни сложности: (вначале делаем первый и если все готово и работает, можно перейти к реализации следующего)
    1. просто вернуть ссылку на статью
    2. ссылку на статью + найденный чанк в текстовом виде
    3. ссылку на статью + найденный чанк в текстовом виде + сгенерированное нейронкой описание статьи
    4. ссылку на статью + найденный чанк в текстовом виде + сгенерированное нейронкой описание статьи + сгенерированное нейронкой резюмирование информации из чанка
    (это как бы может сразу дать ответ на вопрос который задал пользователь, вместо того чтобы он вчитывался в найденный чанк для выуживания из него нужной информации)
